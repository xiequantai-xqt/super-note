# 一、说说你对函数式编程的理解？优缺点？

对函数式编程的理解：

函数式编程将计算视为函数求值，函数可以被传递、返回和作为参数。其核心概念包括：
- **不可变数据**：数据创建后不可修改，操作数据返回新副本。
- **纯函数**：对于相同输入总返回相同输出且无副作用。
- **函数组合**：能将多个函数组合构建复杂功能。

例如，在 JavaScript 中，可以轻松将函数作为参数传递给其他函数实现特定逻辑。

优点：

1. **可维护性高**：
   - 纯函数行为可预测，无外部状态依赖与副作用。
   - 复杂任务可分解为易理解与测试的小函数，修改代码时只需关注相关小函数。

2. **易于测试**：
   - 纯函数仅依赖输入参数，无外部干扰，方便用单元测试工具测试。
   - 函数组合也利于对组合函数进行测试。

3. **并行处理能力**：
   - 因无共享状态与可变数据，适合并行计算。
   - 多线程或分布式环境中各函数实例可独立处理数据，能充分利用多核性能。

4. **代码复用性强**：
   - 纯函数不依赖特定上下文，可在多场景复用。
   - 函数组合也使复用更灵活，能快速构建新功能。

缺点：

1. **性能问题**：
   - 不可变数据结构处理大量或频繁修改数据时，因创建副本会占用更多内存与资源。
   - 函数组合与柯里化等高级特性也可能有额外函数调用开销。

2. **生态系统和工具支持相对较弱**：
   - 相比成熟编程范式，在一些领域工具与库支持不够完善。
   - 新兴技术缺乏足够社区支持与最佳实践案例，应用时开发人员易遇困难。

# 二、解释 JavaScript 严格模式的概念和作用。启用严格模式后，会对代码产生哪些影响？

**JavaScript 严格模式的概念和作用**

JavaScript 严格模式是 ECMAScript 5 引入的一种运行模式，通过在脚本或函数开头添加 `"use strict";` 来启用。

它的主要作用在于：
1. **捕获常见错误**：能捕获常见错误，让开发者更早发现代码问题，比如使用未声明变量这种在非严格模式下可能被忽略的情况，在严格模式下会抛出 `ReferenceError`。
2. **防止不安全操作**：防止不安全操作，如禁止删除不可配置属性，否则会抛出 `TypeError`。
3. **优化引擎性能**：利于 JavaScript 引擎优化，限制一些复杂难优化的语法和行为，提升执行效率。

**启用严格模式后对代码的影响**

启用严格模式后对代码有诸多影响：
- 变量必须先声明再使用；函数参数不能同名，否则抛出 `SyntaxError`；
- 函数内部 `this` 不再默认绑定全局对象，无显式绑定时为 `undefined`；
- `eval()` 有自己的独立作用域，内部声明变量不影响外部；
- 对象字面量属性名不能重复，否则也会抛出 `SyntaxError`。
