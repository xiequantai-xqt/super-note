# 一、常见的状态码

在HTTP协议中，状态码是服务器对客户端请求的响应状态的描述。以下是前端面试中常见的HTTP状态码：

1. **200 OK** - 请求成功，服务器已返回所请求的数据。
2. **201 Created** - 请求成功并且服务器创建了新的资源。
3. **204 No Content** - 请求成功但无内容返回。

4. **301 Moved Permanently** - 资源永久性移除，通常用于重定向到新URL。
5. **302 Found** - 临时性重定向，资源目前从不同位置存取。
6. **304 Not Modified** - 自从上次请求后，请求的页面未被修改过，可以使用缓存版本。

7. **400 Bad Request** - 服务器无法理解请求，通常是客户端错误导致。
8. **401 Unauthorized** - 请求要求用户的身份认证。
9. **403 Forbidden** - 服务器理解请求但拒绝执行，权限不足。
10. **404 Not Found** - 请求的资源在服务器上未找到。

11. **500 Internal Server Error** - 服务器遇到了一个未曾预料的状况，导致了它无法完成对请求的处理。
12. **502 Bad Gateway** - 服务器作为网关或代理，从上游服务器收到无效响应。
13. **503 Service Unavailable** - 服务器暂时不可用，通常是因为过载或维护。
14. **504 Gateway Timeout** - 服务器作为网关或代理没有及时从上游服务器获得响应。

# 二、同样是重定向，307、303、302的区别

302、303 和 307 状态码都是用来指示HTTP重定向的，但它们之间有一些关键的区别，特别是在处理请求方法和实体主体时。下面是这些状态码的详细解释：

1. **302 Found**
   - 这个状态码表示请求的资源临时从不同的URL响应。客户端应当继续使用原有的方法（GET 或 POST）来请求新的URL。
   - 但是，在实践中，许多浏览器在接收到302响应后会将POST请求转换为GET请求，这是为了与早期版本的HTTP规范保持兼容，这种行为并不是RFC标准所规定的。

2. **303 See Other**
   - 当服务器希望用户代理在另一个URI上发起一个新的请求以完成操作时，应该返回此状态码。
   - 客户端发出的原始请求可以是任何类型的，但是接收到303响应后，它应该以GET方式请求新的URI，且不应发送消息体。
   - 通常用于POST之后的重定向，以便用户能够刷新页面而不重复提交数据。

3. **307 Temporary Redirect**
   - 表示请求的资源临时从不同的URL响应，并且未来的请求仍然应使用原有URL。
   - 与302不同的是，307严格要求客户端维持原始的HTTP方法（如POST），即使原请求是POST，重定向后的请求也应该是POST。
   - 该状态码是在HTTP/1.1中引入的，旨在解决302被错误实现的问题。

总结来说：
- 使用302时，虽然规定了应该保持请求方法不变，但实际上很多浏览器会将POST转为GET。
- 303明确地指示客户端用GET方法去请求新位置，并且不应该包含消息体。
- 307则确保了请求方法不会改变，保证了POST请求会被重定向到新位置而不会变成GET请求。

因此，在选择使用哪个状态码时，应该根据实际需求和预期的行为来决定。如果需要确保POST方法不被改变，那么应该使用307；如果希望用户在重定向后能安全地刷新页面而不重复提交数据，则应该使用303。
